### 模块职责

| 模块 | 职责 | 实现方式 |
|------|------|---------|
| Scheduler | 定时触发采集 Pipeline | APScheduler / 系统 Cron |
| Collector Agent | 从各平台拉取原始内容 | 平台 API + 备用爬虫 |
| Filter Agent | 去噪、去重、质量评分 | 规则引擎 + LLM 评分 |
| Summarizer Agent | 生成结构化摘要和简报 | LLM API 调用 |
| Delivery Agent | 推送到飞书 / 输出文件 | 飞书 Webhook |
| Feedback Loop | 收集反馈、更新偏好权重 | 飞书交互消息 + SQLite |

协作流程：Scheduler 按 cron 触发 → Collector 采集 → Filter 过滤 → Summarizer 生成简报 → Delivery 推送。用户反馈异步回流到 Feedback Loop，更新下一轮的过滤权重。

## 2. 数据采集方案

### Twitter/X

- **采集方式**：Twitter API v2（Bearer Token），按关键词 + 关注列表拉取近 24h 推文
- **技术限制**：免费 tier 每月仅 1500 条读取；API 访问政策收紧，第三方限制多
- **应对方案**：
  - 优先使用 API 免费额度，精准设置查询条件减少浪费
  - 备选：通过 Nitter 等开源前端做轻量抓取（需注意合规风险）
  - 按关注的 KOL 列表定向拉取，而非全量搜索

### YouTube

- **采集方式**：YouTube Data API v3，`search.list` 按关键词搜索 + `videos.list` 获取详情
- **技术限制**：每日 10,000 units 配额（一次 search 消耗 100 units）；无法直接获取完整字幕
- **应对方案**：
  - 合理规划 API 调用：每个领域关键词 1 次搜索，每日约 3-5 次搜索 = 300-500 units，远低于配额
  - 字幕获取：使用 `youtube-transcript-api` 库拉取自动生成字幕，作为内容分析的输入
  - 用视频 description + transcript 前 2000 字作为 LLM 输入，平衡成本和信息量

### 微信公众号

- **采集方式**：无官方公开 API，需要间接方案
- **技术限制**：微信生态封闭，反爬严格，内容在微信内网渲染
- **应对方案**：
  - 方案 A（推荐）：接入 RSS 聚合服务（如 WeRSS、feeddd），将目标公众号转为 RSS 源
  - 方案 B：使用搜狗微信搜索作为入口，但稳定性差
  - 方案 C：通过企业微信 API + 自建公众号消息转发，成本较高但最稳定
  - 冷启动阶段建议用方案 A，成本低、接入快

## 3. "高质量"定义与过滤策略

### 冷启动阶段（规则过滤）

采用多维度评分机制，总分 0-100，阈值 40 分以上进入简报：

| 维度 | 权重 | 规则 |
|------|------|------|
| 来源权威性 | 25% | 预设 KOL 白名单（知名创始人、顶级 VC）加分；营销号黑名单扣分 |
| 内容长度 | 15% | Twitter > 50 字、YouTube > 5 分钟、公众号 > 800 字（过短大概率是水文） |
| 互动指标 | 20% | 点赞/转发/评论数超过该账号历史均值的 1.5 倍 |
| 关键词匹配 | 20% | 命中 AI/芯片/机器人领域关键词密度 |
| 时效性 | 10% | 24h 内发布的内容加分 |
| 原创性 | 10% | 去重后的内容（SimHash 相似度 < 0.85）加分 |

**广告/水文过滤规则**：
- 包含推广链接（bit.ly 短链、带 UTM 参数）→ 标记为广告
- 正文含"限时优惠""免费领取""点击关注"等营销关键词 → 降权
- 同一来源 24h 内发布 > 10 条 → 疑似批量发布，降权

### 有反馈后的迭代策略

1. **短期调整**：用户标记"有用"的内容，提取其来源、主题、关键词，动态调整对应维度权重
2. **中期学习**：积累 50+ 条反馈后，训练轻量分类器（如 logistic regression），输入特征为上述维度 + LLM embedding 相似度
3. **长期优化**：用 LLM 做 few-shot 质量判断，prompt 中注入用户历史偏好的正负样本

反馈信号设计：
- 飞书简报中每条内容附带 👍/👎 按钮
- 👍 = 正样本，权重 +1；👎 = 负样本，权重 -2（负反馈信号更强）
- 每周自动生成偏好报告，展示学习效果

## 4. 简报格式设计

### 设计原则

- **5 分钟可读完**：控制在 8-12 条精选内容，每条摘要 2-3 句话
- **手机友好**：短段落、emoji 分隔、无复杂表格
- **结构化**：按领域分区，重要内容置顶
- **可操作**：每条附原文链接，方便深入阅读

### 设计理由

1. **按领域分区**：用户关注 AI/芯片/机器人三个领域，分区浏览效率最高，可以跳过不感兴趣的板块
2. **来源标识**（emoji + 平台名）：一眼看出信息来源，建立可信度判断
3. **2-3 句摘要**：第一句是核心事实，第二句是关键数据/引用，第三句是影响分析——信息密度最大化
4. **底部数据统计**：让用户感知过滤力度（"147 条里只选了 10 条"），建立信任
5. **趋势洞察**：跨内容的 pattern 识别，这是 Agent 相比人工阅读的核心增值点
6. **控制在 10 条**：按平均每条 30 秒阅读，10 条 = 5 分钟，符合需求约束
